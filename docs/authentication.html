Authentication

Fluentax uses the industry standard OAuth 2.0 protocol for authentication. All APIs require a valid access token to be present in the request authorization header.

note

Upon registration you will receive a dedicated service account with a client_id and client_secret pair. You must use these credentials when integrating your system with our APIs.

1. Acquire an access token#

You must acquire your access token from the Fluentax Single Sign-on (SSO) token endpoint:

ParameterValue
grant_typeclient_credentials
client_idyour client id
client_secretyour client secret
scopethe API scope e.g. fx_api
curl https://sso.fluentax.com/auth/realms/fluentax/protocol/openid-connect/token \  -H "Content-Type: application/x-www-form-urlencoded"\  --data "grant_type=client_credentials"\  --data "client_id=<your client_id>"\  --data "client_secret=<your client_secret>"\  --data "scope=<your API scope>"

Response:

{    "access_token": "ey07NzjK3XUMx1wN2pig4gHA",    "expires_in": 300,    "refresh_expires_in": 0,    "token_type": "Bearer",    "not-before-policy": 0,    "scope": "fx_api"}
note

Every access token is valid only for 5 minutes. After the access token expires, you must request a new access token. Refresh tokens are not supported for service accounts.

tip

Tip: You can find a working sample application written in C# in the api-samples repository that demonstrates the authentication flow.

caution

You must ensure that your credentials are never exposed. DO NOT put your credentials in frontend code or anywhere else where they could get compromised. If you think that your credentials were exposed, report it immediately and request a reset.

2. Access an API endpoint#

When accessing an API endpoint, you must specify the access_token from the token endpoint response in the request header.

Get the bank details of AECB:

curl https://fx-api.fluentax.com/banks/AECB \  -H "Authorization: Bearer <your access token>"
